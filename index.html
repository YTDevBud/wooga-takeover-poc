Vulnerability Summary

A development debug endpoint (/mock_webview) is exposed on production at account.booking.com. Android App Links (handle_all_urls) intercepts all URLs on this domain and passes them — including attacker-crafted query parameters — directly to the native Booking.com app. An attacker can inject arbitrary auth_tokens into the app via a single crafted link.

Attack 1 — Android App Link Injection (PRIMARY)

Opens account.booking.com/mock_webview with attacker-controlled auth_tokens. On Android with the Booking.com app installed, the OS intercepts the URL and passes it (with attacker params) to the native app via Intent.

Expected on Android: Booking.com app opens. The app receives auth_tokens=PoC_ATTACKER_TOKEN in the Intent data.
Expected on Desktop: Page opens in browser showing debug page.

Attack 2 — Debug Page + Deep Link (Desktop/Browser)

If tested in a desktop browser or the app doesn't intercept, the debug page renders with the "Exit WebView page" button containing a booking://close_webview deep link with injected tokens.

Attack 3 — postMessage Origin Bypass

Opens the sign-in page as a popup and sends iam-ifr-email with an attacker email. The handler has no origin check.

Technical Evidence: Android App Links

account.booking.com/.well-known/assetlinks.json grants handle_all_urls to the Booking.com Android app:

{
  "relation": [
    "delegate_permission/common.get_login_creds",
    "delegate_permission/common.handle_all_urls"
  ],
  "target": {
    "namespace": "android_app",
    "package_name": "com.booking",
    "sha256_cert_fingerprints": [
      "6A:C8:C8:A9:E6:F3:6E:14:E5:2A:0A:05:15:A3:6B:70:..."
    ]
  }
}

Result: ANY URL on account.booking.com is intercepted by the Android app. The full URL — including query params like ?auth_tokens=EVIL — arrives in the app's Intent as Intent.getData().
