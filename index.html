Vulnerability Summary

A development debug endpoint (/mock_webview) is exposed on the production account.booking.com domain. The page explicitly states "This is a Mock page for WebView demo, it should not be exposed on production."

The page accepts attacker-controlled auth_tokens and context URL parameters, which get injected into a booking://close_webview deep link. When a victim taps the "Exit WebView page" button, the Booking.com native app opens and receives the attacker-controlled tokens. Additionally, the page leaks sensitive tokens (as_token, captcha_access_key, jset, xState) in the HTML source to any unauthenticated visitor.

Step 1 — Open Debug Page with Injected Tokens

This opens the exposed debug page with attacker-controlled auth_tokens injected into the URL. The page loads in the browser.

What happens: The /mock_webview debug page loads. It renders an "Exit WebView page" button whose href contains:
booking://close_webview?context=ATTACKER_CTX&auth_tokens=ATTACKER_TOKEN&next_step=STEP_SUCCESS

Step 2 — Tap "Exit WebView page" Button

On the mock_webview page that opened, tap the "Exit WebView page" button.

What happens on mobile (with Booking.com app):
The booking:// deep link fires. The Booking.com native app opens and receives the attacker-injected auth_tokens value via the deep link handler.

What happens on desktop:
The browser shows a protocol handler prompt for booking://.
Attack scenario: An attacker sends a victim a link like https://account.booking.com/mock_webview?auth_tokens=EVIL via email/SMS/chat. The victim sees a legitimate Booking.com page. They tap "Exit WebView page" and the native app opens with the attacker's injected tokens.

Step 3 — Verify Token Leakage (Desktop)

The leaked tokens are in the HTML source code of the mock_webview page (not visible on this PoC page). To verify:

  1. Open https://account.booking.com/mock_webview in a desktop browser
  2. Press Ctrl+U (View Page Source)
  3. Press Ctrl+F and search for booking.env
  4. You will see the JSON object containing all leaked tokens

On mobile: long-press the URL and choose "Open in new tab", then use the browser menu → "View page source" or add view-source: before the URL.
